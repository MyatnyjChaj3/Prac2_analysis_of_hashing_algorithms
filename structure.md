
# Структура проекта

## 1. Принцип работы

Программа работает в 2 этапа:

1. **Словарная атака** (dictionary attack) — перебирает пароли из существующего списка паролей, чем больше словарь, тем лучше по поиску, но увеличивает время поиска

2. **Полный перебор** (bruteforce) — перебираются все варианты


## 2. Как работает код

### `main.py`
- Проходит по каждому алгоритму (SHA-1, MD5, bcrypt, Argon2)
- Сначала запускает `dictionary_attack()`
- Если остались не найденные хэши — выполняет bruteforce 

Функция dictionary_attack(algorithm, targets) - функция читает словари, создает задачи для каждого слова и отправляет их в пул процессов.

Функция run_bruteforce(algorithm, targets, max_length). Эта функция запускает полный перебор символов, используя многопроцессорность для распределения нагрузки по диапазонам.


### `worker.py` — проверка одного кандидата
- Принимает слово из словаря
- Считает хэш (или проверяет bcrypt/Argon2)
- Возвращает пароль при совпадении

Функция try_dictionary_word(args) (Словарная атака). Ключевой момент — распаковка аргументов. В пуле процессов функция принимает один кортеж (args), который затем распаковывается

Функция check_password(args) (Полный перебор). Эта функция выполняет перебор в заданном чанке (диапазоне) для своей длины и алфавита.

### `config.py`
- Список целевых хэшей из задания
- Путь к папке `dictionaries/`
- Алфавит для bruteforce
